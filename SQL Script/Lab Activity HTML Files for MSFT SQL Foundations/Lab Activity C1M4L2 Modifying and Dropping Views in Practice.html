Modifying and Dropping Views in Practice

Overview

Refine your SQL skills by working with existing views in the Chinook database. You’ll practice updating a view to reflect new business logic and removing a view that’s no longer needed—essential tasks in maintaining a clean, effective database structure.

Learning Outcomes

By the end of this activity, you will be able to:

  • Modify SQL views to adapt to evolving business requirements.
  • Drop SQL views that are no longer needed.
  • Improve the accuracy and performance of your database environment.

Getting Started

Open VS Code, and connect to the Chinook database using the SQL Server. Right click on the Chinook connection to create a new query.

Tasks

Task 1: Create a View for Customer Invoice Summaries

Objective: Create a view summarizing invoice activity by customer.

Steps:

  1. Using the following starter code, write a query to create a view summarizing invoice activity by customer including:
    • Customer First Name
    • Customer Last Name
    • Total Number of Invoices
    • Total Amount Spent
  2. The data should come from the Customer and Invoice tables and be grouped by the customer’s first and last name.
-- 1. Create a view named 'View_CustomerInvoices' that summarizes customer invoice data.

-- 2. Select the following columns to be included in the view:
--    a. Customer's first name as an alias.
--    b. Customer's last name as an alias.
--    c. Total number of invoices for each customer as an alias.
--    d. Total amount spent by each customer as an alias.

-- 3. Use a JOIN clause to combine data from the 'Customer' table and the 'Invoice' table.

-- 4. Group the results by the customer's first and last name to aggregate the data for each customer.

Test Your Work:

SELECT * FROM View_CustomerInvoices;
ORDER BY -- Select the aliased columns.

Expected Output: A list with columns showing customer names, total number of invoices, and total amount spent.

Task 1 Solution:

CREATE VIEW View_CustomerInvoices AS
SELECT c.FirstName AS CustomerFirstName,
    c.LastName AS CustomerLastName,
    COUNT(i.InvoiceId) AS TotalInvoices,
    SUM(i.Total) AS TotalAmountSpent
FROM Customer c
JOIN Invoice i ON c.CustomerId = i.CustomerId
GROUP BY c.FirstName, c.LastName;
GO

SELECT * FROM View_CustomerInvoices
ORDER BY CustomerFirstName, CustomerLastName;
GO

Task 2: Modify the View to Focus on Companies (Exclude NULLs)

Objective: Update the view to focus on companies and exclude records where the Company field is null.

Steps:

  1. Use the ALTER VIEW statement to update View_CustomerInvoices.
  2. Add a WHERE clause to exclude NULL values in the Company column.
  3. Replace the grouping by first/last name with grouping by the company column.
-- 1. Alter the existing view 'View_CustomerInvoices' to focus on company data.

-- 2. Select the following columns to be included in the view:
--    a. Customer's company name as an alias.
--    b. Total number of invoices for each company as an alias.
--    c. Total amount spent by each company as an alias.

-- 3. Use a JOIN clause to combine data from the 'Customer' table and the 'Invoice' table.

-- 4. Exclude rows where the company name is NULL.

-- 5. Group the results by the company name to aggregate the data for each company.

Test Your Work:

SELECT * FROM View_CustomerInvoices
ORDER BY -- Select the aliased column.

Expected Output: A list of companies with columns showing the total number of invoices and total amount spent.

Task 2 Solution:

ALTER VIEW View_CustomerInvoices AS
SELECT c.Company,
    COUNT(i.InvoiceId) AS TotalInvoices,
    SUM(i.Total) AS TotalAmountSpent
FROM Customer c
JOIN Invoice i ON c.CustomerId = i.CustomerId
WHERE c.Company IS NOT NULL
GROUP BY c.Company;
GO

SELECT * FROM View_CustomerInvoices
ORDER BY TotalAmountSpent DESC;
GO

Common Issues & Solutions

  • Problem: Errors in modifying the view
    Solution: Double-check your ALTER VIEW syntax and ensure all columns used in the view are properly referenced.
  • Problem: Unable to drop the view
    Solution: Ensure the view name is correct and that you have sufficient permissions.

Summary

Modifying and dropping SQL views are key skills in maintaining an effective database structure. This activity has provided practice in adapting views to meet new business needs and cleaning up outdated structures.

Key Points

  • Dropping views removes obsolete structures and clears up database space.
