Use Views in Practice

Overview

In this activity, you will explore the concept of views to simplify complex query logic and enhance data presentation. By applying these techniques in a real-world context with the Chinook Database, you will refine your skills and develop the ability to construct queries that are both efficient and easy to understand.

Learning Outcomes

By the end of this activity, you will be able to:

  • Create SQL views to summarize data.
  • Consider how views help manage complex, repeated queries.
  • Extract relevant insights from a view.

Getting Started

Open VS Code, and connect to the Chinook database using the SQL Server. Right click on the Chinook connection to create a new query.

Tasks

Task 1: Design a Customer Spending Overview View

Objective: Create a view which summarizes simple data.

Steps:

  1. Copy the following starter code and paste it into a new query.
  2. Complete the view summarizing customer invoice data using the Customer and Invoice tables. The results should show the customer's first name, last name, the total number of invoices, and the total amount spent for each customer.
  3. Group the view by first and last name in ascending order.
-- This code creates a SQL View named 'View_CustomerInvoices' that provides a summary of customer invoice data.
-- 1. Define the view name as 'View_CustomerInvoices' using the CREATE VIEW statement.
-- 2. Select the following columns to be included in the view:
--    a. Customer's first name as an alias.
--    b. Customer's last name as an alias.
--    c. Total number of invoices for each customer using COUNT on InvoiceId, as an alias.
--    d. Total amount spent by each customer using SUM on the 'Total' column, as an alias.
-- 3. Use a JOIN clause to combine data from the 'Customer' table and the 'Invoice' table.
-- 4. Group the results by the customer's first and last name to aggregate the data for each customer.

Tip: Create the SELECT query first and run it to make sure you're getting the right data. Once you're confident, wrap it in a CREATE VIEW statement.

Task 1 Solution:

CREATE VIEW View_CustomerInvoices AS
SELECT c.FirstName AS CustomerFirstName,
    c.LastName AS CustomerLastName,
    COUNT(i.InvoiceId) AS TotalInvoices,
    SUM(i.Total) AS TotalAmountSpent
FROM Customer c
JOIN Invoice i ON c.CustomerId = i.CustomerId
GROUP BY c.FirstName, c.LastName;
GO

Task 2: Design a Detailed Invoice View

Objective: Consider how views help manage complex queries.

Steps:

  1. Use the following starter code to develop a view that encapsulates the invoice details (track, price, unit price, quantity, and total cost) for each country and date. Use the Invoice and InvoiceLine tables.
-- This code creates a SQL View named 'View_InvoiceDetails' that provides detailed information about invoices.
-- 1. Define the view name as 'View_InvoiceDetails' using the CREATE VIEW statement.
-- 2. Select the following columns to be included in the view:
--    a. Invoice ID
--    b. Invoice date
--    c. Billing country
--    d. Track ID
--    e. Unit price
--    f. Quantity of the track purchased.
--    g. The total calculated as from the 'InvoiceLine' table, as an alias.
-- 3. Use a JOIN clause to combine data from the 'Invoice' table and the 'InvoiceLine' table.
-- 4. Join with the 'InvoiceLine' table to get detailed information about each invoice's line items.

Task 2 Solution:

CREATE VIEW View_InvoiceDetails AS
SELECT i.InvoiceId,
    i.InvoiceDate,
    i.BillingCountry,
    il.TrackId,
    il.UnitPrice,
    il.Quantity,
    (il.UnitPrice * il.Quantity) AS LineTotal
FROM Invoice i
JOIN InvoiceLine il ON i.InvoiceId = il.InvoiceId;
GO

Task 3: Query the Views to Extract Information

Objective: Consider how views help manage repeated queries, using views to extract relevant insights.

Steps:

  1. Use the starter code to query the Customer Spending view to find the top 10 spending customers by total amount paid.
    2. -- Query the View_CustomerInvoices view to retrieve the top 10 customers by total amount spent in descending order.
  2. Use the starter code to query the Invoice Details view to retrieve the total amount spent by country and sorted by country in descending alphabetical order.
    3. -- Query the Invoice Details view to retrieve the total amount spent by country, and sorted the by country in descending alphabetical order.

Task 3.1 Solution:

SELECT TOP 10 CustomerFirstName, CustomerLastName, TotalAmountSpent
FROM View_CustomerInvoices
ORDER BY TotalAmountSpent DESC;

Task 3.2 Solution:

SELECT BillingCountry, SUM(LineTotal) AS TotalSpending
FROM View_InvoiceDetails
GROUP BY BillingCountry
ORDER BY TotalSpending DESC;

Summary

In this activity you created and queried views to consider how you might use views to manage complex, repeated tasks and access greater insights. Through practice, reflection, and feedback, you will strengthen your command over advanced querying techniques essential for effective data manipulation.

Key Points

  • Views optimize efficiency and insights by encapsulating data into a simple, reusable format.
  • Views are especially valuable in scenarios where data requires consistent updates or repeated queries.
  • Views simplify the presentation of complex data, making it easier to access specific insights.
