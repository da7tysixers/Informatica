Examine Tables in a Sample Database

This activity is designed to provide you with practical exposure to examining database tables, exploring their structure, relationships, and the role these play in data retrieval and organization. Using the Chinook database, you will gain hands-on experience analyzing tables and their interconnections, enhancing your capability to efficiently work with relational database schemas.

Learning Outcomes

  • Identify the structure, primary keys, and columns of tables in the Chinook database.
  • Explain a one-to-many relationship and its impact on data retrieval and organization.
  • Analyze a junction table illustrating a many-to-many relationship.

Getting Started

Open VS Code, and connect to the Chinook database using the SQL Server extension. Right click on the Chinook connection to create a new query.

  1. Start by launching Visual Studio Code on your machine. Use the Start menu to search for "Visual Studio Code" and press Enter.
  2. Navigate to the SQL Server extension in the side panel using the Tab key. Listen for "SQL Server icon" and press Enter to expand the side panel.
  3. There should be an existing database connection named Chinook. Use the Tab key to navigate to the Chinook connection and press Enter to connect to the database.
  4. Once connected to the Chinook database, you can start exploring the database as outlined in the next tasks.

Tasks

Task 1: Analyze Individual Tables

Objective: Examine the structure of the Employee table, document the data types, primary keys, and foreign keys for the columns, and understand the hierarchy through the ReportsTo field.

  1. Use the SQL Server extension to locate the Employee table. Use Tab to navigate and press Enter to expand it.
  2. Navigate to the Columns tab within the Employee table and listen as the screen reader reads the data types for each column, including whether a column is a primary or foreign key (e.g., EmployeeId, FirstName, LastName, ReportsTo).
  3. Reflect on how the ReportsTo field establishes reporting relationships within the organization.

Task 2: Investigate 'Invoice' and 'Customer' Tables

Objective: Explore how the Invoice and Customer tables relate to each other.

  1. Navigate to the Columns folders for both the Invoice and Customer tables, using the Tab key and Enter key to open them.
  2. Listen to the screen reader as it describes the CustomerId and InvoiceId relationships.
  3. Note how a customer can have multiple invoices, illustrating a one-to-many relationship.

Task 3: Discover Many-to-Many Relationships with Playlists

Objective: Analyze the Playlist, PlaylistTrack, and Track tables to understand many-to-many relationships.

  1. Navigate to the column folders in the Playlist, PlaylistTrack, and Track tables. Use Tab and Enter to explore the columns.
  2. Listen to the screen reader as it explains how the PlaylistTrack table is a junction between the Playlist and Track tables.
  3. Reflect on how a Track can belong to multiple Playlists, demonstrating the concept of many-to-many associations.

Task 4: Examine Invoice Details

Objective: Analyze the Invoice and InvoiceLine tables.

  1. Navigate to the columns folders for both Invoice and InvoiceLine tables. Use the Tab key and Enter key to investigate the columns.
  2. Listen as the screen reader describes the relationship between Invoice, which represents transaction information, and InvoiceLine, which lists each sale item.
  3. Understand how these tables work together to provide detailed sales data.

Summary

In this activity, you explored key structures and relationships within the Chinook database, enhancing your understanding of relational database organization and data retrieval. This foundational knowledge is essential as we will continue to use the Chinook database for learning SQL.

Key Points

Databases are collections of datasets connected through primary keys, one-to-many, and many-to-many relationships.
