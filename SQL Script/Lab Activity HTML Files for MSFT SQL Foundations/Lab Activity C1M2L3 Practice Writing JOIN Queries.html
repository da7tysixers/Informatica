Practice Writing JOIN Queries

Overview

In this activity, you will practice writing JOIN queries using a SQL simulator. The tasks will focus on combining data from multiple tables, such as merging customer and order data to view complete order details. This hands-on exercise will reinforce the concepts learned in the lesson and provide immediate feedback.

Learning Outcomes

By the end of this activity, you will be able to:

  • Write a JOIN query to combine data from two tables.
  • Understand how different types of JOIN queries affect output results.

Getting Started

Open VS Code, and connect to the Chinook database using the SQL Server.

Tasks

Task 1: Combine Customer and Invoice data.

Objective: Write a JOIN query to combine data from the Customer and Invoice tables.

Steps:

  1. Write an INNER JOIN query to retrieve the customer ID, last name and invoice ID on the PostalCode column from the Customer table and the BillingPostalCode on the Invoice table.
  2. Execute the query, look at all the results. Note the number of records and whether there are any NULL values in the results.
SELECT Customer.CustomerId, 
    Customer.LastName, 
    Invoice.InvoiceId
FROM Customer
JOIN -- Your code here to join the Invoice table

Tip: When using JOINs, always check the number of rows returned and whether any columns have NULLs. This helps you identify if data is missing due to the type of JOIN used or issues in relationships between tables.

Task 1 Solution:

-- Task 1: Combine customer and invoice data.
SELECT
    Customer.CustomerId,
    Customer.LastName,
    Invoice.InvoiceId
FROM Customer
JOIN Invoice ON Customer.PostalCode = Invoice.BillingPostalCode;
        
--The Customer table contains NULL values in the PostalCode column. These records are excluded when using an INNER JOIN, resulting in 384 retrieved records (rows) with no records containing NULL.

Task 2: Combine Customer and Invoice data with a LEFT JOIN.

Objective: Try out a LEFT JOIN and analyze the results.

Steps:

  1. Now, write a LEFT JOIN query to retrieve the customer ID, last name and invoice ID on the PostalCode column from the Customer table and the BillingPostalCode on the Invoice table.
  2. Execute the query, look at all the results, and note the number of records and whether there are any NULL values in the results.

Task 2 Solution:

-- Task 2: Explore LEFT JOIN
SELECT
    Customer.CustomerId,
    Customer.LastName,
    Invoice.InvoiceId
FROM Customer
LEFT JOIN Invoice ON Customer.PostalCode = Invoice.BillingPostalCode;

-- The LEFT JOIN will produce 388 records, including four customers without a PostalCode. This is because the LEFT JOIN retrieves all Customer table data and any matching InvoiceId's from the Invoice table. For those four customers without a matching BillingPostalCode, the InvoiceId value is NULL. The following query validates that these four customers don’t have a PostalCode.

SELECT CustomerId, LastName, PostalCode  
FROM Customer
WHERE PostalCode IS NULL;

Task 3: Combine Customer and Invoice Data with a RIGHT JOIN.

Objective: Try out a RIGHT JOIN and analyze the results.

Steps:

  1. Next, change your LEFT JOIN to a RIGHT JOIN and execute the query.
  2. Look at all the results, and note the number of records and whether there are any NULL values in the results.

Task 3 Solution:

-- Task 3: Explore RIGHT JOIN
SELECT
    Customer.CustomerId,
    Customer.LastName,
    Invoice.InvoiceId
FROM Customer
RIGHT JOIN Invoice ON Customer.PostalCode = Invoice.BillingPostalCode;

-- The RIGHT JOIN retrieves all records from the Invoice table and any matching records from the Customer table. It will also include NULL values for CustomerIds and LastNames where there is no match in the Customer table. This is why the RIGHT JOIN returns 412 records, which includes records where the PostalCode from the Customer table is NULL.

Task 4: Combine Customer and Invoice Data with a FULL OUTER JOIN.

Objective: Try out a FULL OUTER JOIN and analyze the results.

Steps:

  1. Finally, change your RIGHT JOIN to a FULL OUTER JOIN and execute the query.
  2. Look at all the results and note the number of records and whether there are any NULL values in the results.

Task 4 Solution:

-- Task 4: Explore FULL OUTER JOIN
SELECT
    Customer.CustomerId,
    Customer.LastName,
    Invoice.InvoiceId
FROM Customer
FULL OUTER JOIN Invoice ON Customer.PostalCode = Invoice.BillingPostalCode;

-- The FULL OUTER JOIN returns all records from both tables, including matching and non-matching ones. In this instance, the FULL OUTER JOIN produced 416 records and includes records containing NULL values from both tables.
